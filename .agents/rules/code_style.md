# Rule: Strict Code Style Rules

## Guardrail Constraints
You MUST follow these strict code style rules in all code modifications:

1. **Avoid Monoliths**: Keep code modular and split logic into focused files rather than giant routers.
2. **No AI Jargon / Cyber Larping**: Talk plainly and avoid cinematic "cyber" or dramatic engineering language (e.g., in logs, comments, or terminal outputs).
3. **Keep Code Text Short**: No long walls of text in warnings, console logs, or commands inside the code. Keep all text scannable and concise.
4. **No Redundant Function Calls**: Do not call the same function multiple times on successive lines; store the value in a variable instead.
5. **No Emojis in Code**: Never insert emojis in strings, comments, or terminal outputs inside the source code.
