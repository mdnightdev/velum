# Rule: Codebase Architecture Audit First

## Guardrail Constraint
Before starting any code modifications or fixing any bugs, you MUST perform a comprehensive codebase audit. Do NOT edit any files until you have mapped the architecture and presented a summary to the user for approval.

## Actionable Steps
1. Map out the project structure and locate core entry points.
2. Read all database wrappers, repository definitions, and API route definitions.
3. Present a summary of the mapped architecture and your proposed modification plan to the user.
4. Obtain explicit approval from the user before writing or modifying any code.
