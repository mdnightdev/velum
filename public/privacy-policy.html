Introduction

Velum ("we," "us," "our") is committed to protecting your privacy. This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use our platform, including any associated services, websites, and applications (collectively, the "Platform").

Please read this Privacy Policy carefully. By accessing or using the Platform, you acknowledge you have read and understood this policy. If you do not agree with our practices, do not use the Platform.

Our Commitment: Velum was built with a security-first architecture. We employ AES-256-GCM encryption at rest, Argon2id password hashing, and cryptographic nonce-based authentication. Your privacy is not an afterthought — it is the foundation of our platform.

1. Information We Collect

1.1 Information You Provide

When you create an account or use the Platform, we collect:

  • Account Credentials: Username, password hash (Argon2id, not plaintext), safe word hash, panic phrase hash, and recovery key hash. We never store plaintext passwords or recovery keys.
  • Profile Information: Display name, bio, avatar image, location (derived from IP).
  • Communications: Messages you send through the Platform, including DM content, lounge messages, and support ticket correspondence.
  • Marketplace Data: Listings you create, transactions you conduct, reviews you submit, and dispute records.
  • Financial Information: Wallet balances, transaction history, payment methods (tokens, not full card numbers), and KYC verification documents (simulated).

1.2 Information Collected Automatically

  • Device Information: Device fingerprint, user agent, screen resolution, browser type.
  • IP Address: Used for geo-location, rate limiting, and security monitoring.
  • Usage Data: Features accessed, rooms joined, messages sent, sessions duration.
  • Session Data: Login timestamps, session duration, activity metrics, idle time.

3. Data Storage & Security

3.1 Encryption

All sensitive data at rest is encrypted using AES-256-GCM (Galois/Counter Mode) with unique initialization vectors. Passwords and other credentials are hashed using Argon2id, the current industry standard for password hashing, with unique per-user salts.

3.2 Storage Locations

Data is stored in a local SQLite database with WAL (Write-Ahead Logging) mode for performance. An encrypted JSON backup file is maintained for redundancy. Cloud backups to PostgreSQL (Neon) may be enabled for disaster recovery.

3.4 Security Measures

  • Nonce-based Authentication: Each login requires a cryptographic nonce to prevent replay attacks.
  • Rate Limiting: Exponential backoff on failed authentication attempts prevents brute force attacks.
  • Idle Timeout: Sessions automatically expire after 30 minutes of inactivity.
  • Panic Protocol: Users can trigger immediate session revocation and account quarantine.
  • File Permissions: Database files are restricted to 0600 (owner read/write only).
